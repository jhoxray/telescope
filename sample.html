Logging & monitoring for Meteor framework since 2012.

Observatory makes your Meteor application development a breeze by providing easy logging API, key application internals monitoring and integrated javascript console right here in the browser. This site is itself a Meteor app and has Observatory installed. See the panel below? Check it out and grab a copy for your project in two easy steps: mrt add observatory in the shell and {{>logs_bootstrap}} </body> in your templates.

Why?

We got frustrated with the need to switch between the terminal and Chrome's js console to keep track of Meteor app logs, plus needed a basic standard logging package that can support different log levels and present all that in one place in a nice format. Then kinda got carried away. If you are thinking "IDE", then yup, that's what we are thinking eventually. Someday. Maybe...

Observatory panel is controlled by pressing the "O!" button in the bottom-left corner of the page: you can have it small, medium, large or completely turned off. By moving between tabs you can analyze logs (updated live), your application's templates with events, as well as monitor other internals (currently, Session state, more to come). Observatory is currently version 0.1.6 and you may find it quite handy in your Meteor app development. We do :)

What does it do?

  • Supports different log levels (currently FATAL, ERROR, WARNING, INFO, VERBOSE) with corresponding methods for message output, optional logging to console, pretty output of both Client and Server logs using Twitter Bootstrap right in the browser.
  • Provides live monitoring of your app's templates with all events (they tend to become too many very fast, so keeping track of all of them in one place is very useful) and some other internals (currently, Session object)

Usage

Installation is as easy asmrt add observatory. Also feel free to browse the code for this app.

For details, see Observatory package on GitHub, but in a nutshell you simply call
TL = TLog.getLogger(TLog.LOGLEVEL_MAX,true) once in your app's common code (that gets executed on both client and server) and then call the following obvious methods: TL.fatal("..."), error, warn, info, verbose.

To display the main Observatory panel, just plugin logs_bootstrap Handlebars template anywhere in your html pages, preferrably right before closing "body" tag.
